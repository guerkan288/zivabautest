── COOKIE-BANNER: WIE ER FUNKTIONIERT ───────────────────────────────

Die Website setzt von sich aus KEINE Cookies und lädt ohne Zustimmung
NICHTS von Drittanbietern nach. Der einzige "kritische" Programmpunkt
sind die drei Google-Schriftarten (Bricolage Grotesque, Plus Jakarta
Sans, IBM Plex Mono) — deren Server-Verbindung überträgt beim Laden
automatisch die IP-Adresse des Besuchers an Google (Urteil LG München,
20.01.2022, Az. 3 O 17493/20). Deshalb:

  • Beim ersten Besuch erscheint unten ein Banner mit drei Optionen:
    "Einstellungen" / "Nur notwendige" / "Alle akzeptieren"
  • Nur bei "Alle akzeptieren" (oder im Modal aktiviert) wird die
    Google-Fonts-Verbindung tatsächlich aufgebaut
  • Bei "Nur notwendige" bzw. ohne Reaktion bleibt die Seite mit einer
    optisch sehr ähnlichen Systemschrift (z. B. Georgia/Segoe UI) —
    komplett ohne Verbindung zu Google
  • Die getroffene Wahl wird im Local Storage des Browsers gespeichert
    (kein Cookie im klassischen Sinn, technisch notwendig, daher ohne
    weitere Einwilligung zulässig) — der Banner erscheint dadurch nicht
    bei jedem Besuch erneut
  • Über den Link "Cookie-Einstellungen" im Footer kann die Wahl
    jederzeit geändert werden

OPTION: Google Fonts komplett lokal hosten (empfohlene Königsklasse)
Rechtlich noch sauberer als der Cookie-Banner ist es, die Schriften
direkt auf dem eigenen Server zu speichern — dann entsteht überhaupt
keine Verbindung zu Google und ihr braucht für die Schriftarten gar
keinen Cookie-Hinweis mehr. So geht's:
  1. Auf https://fonts.google.com die drei Schriften herunterladen:
     "Bricolage Grotesque", "Plus Jakarta Sans", "IBM Plex Mono"
  2. Die .ttf-Dateien in assets/fonts/ ablegen
  3. In assets/style.css ganz oben @font-face-Regeln ergänzen, z. B.:
       @font-face {
         font-family: 'Plus Jakarta Sans';
         src: url('fonts/PlusJakartaSans-Regular.ttf') format('truetype');
         font-weight: 400; font-display: swap;
       }
     (für jeden benötigten Schriftschnitt eine eigene Regel)
  4. In script.js den kompletten Cookie-Banner-Block entfernen und in
     index.html/impressum.html/datenschutz.html die Banner- und
     Modal-<div>-Blöcke löschen
  5. In datenschutz.html den Abschnitt "Externe Schriftarten (Google
     Fonts)" entfernen bzw. durch einen Satz ersetzen wie: "Wir nutzen
     lokal gehostete Schriftarten, es fließen keine Daten an Dritte."
Diesen Schritt könnt ihr jederzeit später nachholen, sobald ihr Zeit
dafür habt — bis dahin ist der eingebaute Cookie-Banner eine solide,
rechtlich abgesicherte Zwischenlösung.

 🆕 NEUE FUNKTIONEN: WHATSAPP, GOOGLE-BEWERTUNGEN & PROJEKTE
═════════════════════════════════════════════════════════════════════

── WHATSAPP-BUTTON (grüner Button, unten rechts auf jeder Seite) ──────

Aktuell verlinkt der Button auf eine Platzhalter-Nummer. So trägt ihr
eure echte Nummer ein:
  1. Öffnet index.html, impressum.html, datenschutz.html und
     projekte.html im Editor
  2. Sucht jeweils nach: 491234567890
  3. Ersetzt das durch eure WhatsApp-Nummer im internationalen Format
     OHNE +, 00 oder Leerzeichen — z. B. wird aus 0151 234567 die
     Zeichenfolge 49151234567
  4. Optional: den Text hinter "?text=" in der gleichen Zeile anpassen
     — das ist die Nachricht, die beim Öffnen im Chat vorausgefüllt ist

── GOOGLE-BEWERTUNGEN (Karte über dem Kontaktformular) ─────────────────

Zeigt aktuell einen Platzhalter (4,9 von 5, als solcher gekennzeichnet).
Sobald euer Google-Unternehmensprofil eingerichtet ist:
  1. In index.html nach "google-reviews-card" suchen
  2. "4,9 von 5" durch eure echte Bewertung ersetzen
  3. Den Link bei "Auf Google ansehen" (aktuell google.com/maps) durch
     euren echten Profil-Link ersetzen — zu finden über euer
     Google-Unternehmensprofil unter "Profil teilen"
  4. Für eine Anzeige, die sich automatisch aktualisiert (statt eine
     Zahl von Hand zu pflegen), gibt es Widget-Dienste wie Elfsight,
     EmbedSocial oder Trustindex — die liefern ein fertiges Code-
     Snippet mit eurer Google-Place-ID zum Einfügen an gleicher Stelle

── PROJEKTE-SEITE (eigene Seite, nur über das Menü erreichbar) ────────

projekte.html zeigt aktuell "Diese Seite ist noch im Aufbau", damit
ihr sie erst mit echten, abgeschlossenen Projekten befüllt, wenn ihr
soweit seid. So aktiviert ihr eure erste Projekt-Galerie:
  1. projekte.html im Editor öffnen, nach "projects-empty" suchen
  2. Diesen Block löschen oder mit <!-- und --> auskommentieren
  3. Direkt darunter liegt ein bereits fertiger, nur auskommentierter
     Beispiel-Block mit 3 Projekt-Karten (Suchbegriff: BEISPIEL-KARTEN)
     — die Kommentarmarkierung am Anfang und Ende dieses Blocks
     entfernen, schon ist er aktiv
  4. Pro Karte: Bildpfad (assets/images/...), die kleine Kategorie-
     Beschriftung (project-tag) und den Titel an euer echtes Projekt
     anpassen. Für weitere Projekte einfach eine ganze
     <article class="project-card"> ... </article> duplizieren — das
     Raster passt sich automatisch an, auch bei 5, 6 oder mehr Karten

 🚀 HOCHLADEN ZU IONOS (mit FTP)
═════════════════════════════════════════════════════════════════════

1. Hosting aktivieren / FTP-Zugang einrichten
   - Melde dich bei ionos.de an
   - Gehe zu "Hosting" → Dein Hosting-Paket
   - Aktiviere FTP (oder "SFTP" für sicherer)
   - Notiere dir:
     * FTP-Server (z.B. ftp.deinedomain.de)
     * Benutzername (z.B. u123456789)
     * Passwort

2. FTP-Client herunterladen
   - Kostenlos: FileZilla (https://filezilla-project.org)
   - Oder: WinSCP (Windows)

3. FileZilla einrichten
   - Öffne FileZilla
   - "Datei" → "Servermanager"
   - Klicke "Neuer Server"
   - Gib ein:
     * Host: dein FTP-Server
     * Benutzer: dein Benutzername
     * Passwort: dein Passwort
   - Verbinden

4. Dateien hochladen
   - Im rechten Fenster navigiere zu: public_html/ (oder www/)
   - Ziehe die Ordnerstruktur dort rein:
     * index.html
     * impressum.html
     * datenschutz.html
     * assets/ (kompletter Ordner mit CSS, JS, Bilder)

5. Fertig!
   - Öffne http://deinedomain.de im Browser
   - Website sollte live sein

 🆗 KONTAKTFORMULAR KONFIGURIEREN
═════════════════════════════════════════════════════════════════════

Das Formular im Footer funktioniert lokal (zeigt "Danke"-Meldung an),
aber versendet derzeit KEINE E-Mails.

Um echte E-Mail-Benachrichtigungen zu bekommen, brauchst du einen Service:

OPTION 1: Formspree (einfach, kostenlos)
──────────────────────────────────────────
1. Gehe zu https://formspree.io
2. Registriere dich (kostenlos)
3. Erstelle ein neues Projekt für deine Domain
4. Du bekommst eine "Action URL", z.B.:
   https://formspree.io/f/xyzabc123

5. Öffne index.html im Editor
6. Suche nach: <form id="contactForm"
7. Ersetze die Action:
   VORHER: (action="" ist leer)
   NACHHER: action="https://formspree.io/f/xyzabc123" method="POST"

8. Speichern und neu hochladen
9. Fertig! Formulare kommen jetzt per E-Mail

OPTION 2: PHP auf dem Hoster nutzen
────────────────────────────────────
Falls dein Hoster PHP unterstützt (alle modernen Hoster tun das):
1. Erstelle eine neue Datei: mailer.php
2. Code (Beispiel):
   <?php
   if ($_SERVER['REQUEST_METHOD'] == 'POST') {
     $name = htmlspecialchars($_POST['name']);
     $email = htmlspecialchars($_POST['email']);
     $subject = htmlspecialchars($_POST['topic']);
     $message = htmlspecialchars($_POST['message']);
     
     $to = "info@ziva-bau.de";
     $headers = "From: " . $email;
     $body = "Name: $name\nE-Mail: $email\nThema: $subject\n\n$message";
     
     mail($to, $subject, $body, $headers);
     echo "sent";
   }
   ?>

3. In index.html, in der Form:
   action="mailer.php" method="POST"

4. Hochladen (PHP-Datei und index.html)
5. Testen!

 ❓ HÄUFIG GESTELLTE FRAGEN
═════════════════════════════════════════════════════════════════════

F: Die Website lädt sehr langsam?
A: Typisch bei der ersten Ladung wegen der Bilder. Danach ist es
   schneller (Browser-Cache). Falls persistent langsam: Frag deinen
   Hoster nach Speicher/CPU oder nutze einen CDN.

F: Formular zeigt Error?
A: 1) Hast du die action="..." in der Form gesetzt?
   2) Ist mailer.php auf dem Server?
   3) Prüfe die Dateirechte (644 für .html, .jpg, .css)

F: Bilder zeigen nicht?
A: 1) Prüf, dass assets/images/ komplett hochgeladen ist
   2) Achte auf Groß-/Kleinschreibung (Dateinamen!)
   3) Öffne Browser-Konsole (F12) → Errors prüfen

F: Kann ich die Website offline nutzen?
A: Ja, download alle Dateien und öffne index.html im Browser.

F: Kann ich die Website auf Handy bearbeiten?
A: Ja, benutze deinen Dateimanager beim Hoster, um Dateien zu bearbeiten.
   Oder nutze einen FTP-Client auf dem Handy (FileSafe, etc.)
